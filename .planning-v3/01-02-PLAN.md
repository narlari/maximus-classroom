# Plan 01-02: Classroom UI + tldraw Integration

## Overview
Build the V3 classroom page: a tldraw whiteboard with teacher shapes rendered on it, and a minimal controls panel. This is the first thing Sung will actually see.

## Context
- tldraw v4 spike is at src/components/whiteboard-spike/TldrawSpike.tsx — reference it for correct tldraw v4 imports
- V3 lesson generation is done: import from src/lib/classroom-v3
- board-renderer.ts has renderProblemToBoard(editor, problem) — use it
- Route is /classroom-v3 (separate from old /classroom/[studentId] — don't touch V1/V2)
- Use 'use client' for components that need tldraw (it's a client-side library)
- tldraw v4 import: `import { Tldraw, Editor } from 'tldraw'` and `import 'tldraw/tldraw.css'`

## Implementation

### Step 1: Page (src/app/classroom-v3/page.tsx)
Simple server component that renders ClassroomV3.

```tsx
import ClassroomV3 from '@/components/classroom-v3/ClassroomV3'

export default function ClassroomV3Page() {
  return <ClassroomV3 />
}
```

### Step 2: ClassroomV3 root component (src/components/classroom-v3/ClassroomV3.tsx)
- 'use client'
- State: `lesson: LessonPlan | null`, `currentProblemIndex: number`, `phase: ClassroomPhase`, `loading: boolean`
- Holds a ref to the tldraw Editor: `editorRef = useRef<Editor | null>(null)`
- Renders: `<ClassroomWhiteboard>` (70% height) + `<LessonControls>` (30% height)
- `handleStartLesson()`: sets loading=true, fetches POST /api/lesson/generate with topic="addition and subtraction" and gradeLevel=4, sets lesson state, calls renderProblemToBoard(editor, lesson.problems[0]), sets phase="teaching"
- `handleNextProblem()`: increments index, calls renderProblemToBoard for next problem
- Full viewport height layout (h-screen, flex col)

### Step 3: ClassroomWhiteboard (src/components/classroom-v3/ClassroomWhiteboard.tsx)
- 'use client'
- Props: `onEditorReady: (editor: Editor) => void`
- Renders `<Tldraw>` with `onMount={(editor) => onEditorReady(editor)}`
- Hide the tldraw toolbar/menu for a cleaner classroom look:
  - Use `components` prop: `<Tldraw components={{ Toolbar: null, MainMenu: null, NavigationPanel: null }}>`
- The tldraw CSS must be imported here or in a parent

### Step 4: LessonControls (src/components/classroom-v3/LessonControls.tsx)
- 'use client'
- Props: `phase: ClassroomPhase`, `lesson: LessonPlan | null`, `currentProblemIndex: number`, `loading: boolean`, `onStartLesson: () => void`, `onNextProblem: () => void`
- Shows:
  - When phase=idle: "Start Lesson" button
  - When loading: "Generating lesson..." spinner
  - When phase=teaching or student_turn: current problem text (lesson.problems[currentProblemIndex].problemText)
  - When phase=teaching: "Next Problem" button (if not last problem)
  - When phase=ended: "Lesson complete!"
- Keep it simple and clean — styled with Tailwind

## Verification
1. `npm run build` passes
2. Navigate to /classroom-v3 — see tldraw whiteboard + control panel
3. Click "Start Lesson" — loader appears, then teacher shapes appear on the board
4. Teacher shapes are not moveable/selectable by user (isLocked)
5. Student can still draw their own shapes on the canvas
6. "Next Problem" clears teacher shapes and draws next problem

## Notes
- The topic is hardcoded for now ("addition and subtraction", grade 4) — we'll make it configurable later
- Don't style it heavily — clean and functional is the goal. Tailwind is fine.
- Don't touch any V1/V2 components. New files only.
- If tldraw's isLocked doesn't prevent selection, try also setting `meta: { locked: true }` or use editor.updateShape to set `isLocked` after creation.
