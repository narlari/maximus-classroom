What the PRD says about vision:
- GPT-4o evaluates student whiteboard work
- Input: screenshot of student drawing area + current lesson step context
- Output: { correct: boolean, feedback: string, hint?: string }
- Feedback drives what the AI says next (via TTS) and whether lesson advances

## Deliverables

### 1. `src/lib/classroom-v3/vision-eval.ts`
```typescript
export interface EvaluationResult {
  correct: boolean
  feedback: string  // What tutor says to student
  hint?: string     // Optional hint if incorrect
  confidence: 'high' | 'medium' | 'low'
}

export async function evaluateStudentWork(
  screenshotDataUrl: string,  // base64 PNG from tldraw
  lessonStep: LessonStep,     // current step context
  subject: string,            // e.g. "4th grade math"
): Promise<EvaluationResult>
```

Implementation:
- Call OpenAI GPT-4o via `openai.chat.completions.create` with vision
- System prompt: focused evaluator for elementary math/reading
- Pass the screenshot + step context (what was asked, expected answer/approach)
- Return structured JSON (use response_format: { type: 'json_object' })
- Handle errors gracefully — return { correct: false, feedback: "Let me take another look...", confidence: 'low' }

### 2. `src/app/api/lesson/evaluate/route.ts`
POST endpoint:
```typescript
// Request body:
{ screenshotDataUrl: string, step: LessonStep, subject: string }
// Response:
EvaluationResult
```

### 3. `src/lib/classroom-v3/index.ts` — export evaluateStudentWork

### 4. `src/components/classroom-v3/ClassroomV3.tsx` — Add "Check My Work" button
- Button in the classroom UI (below whiteboard or in LessonControls)
- On click: capture tldraw canvas screenshot → POST to /api/lesson/evaluate → display result
- Show feedback text in UI + speak it via speakText()
- If correct: show ✅ and enable "Next Step" 
- If incorrect: show feedback + hint, keep on same step

## tldraw Screenshot Capture

tldraw v4 has `editor.getSvgElement()` or `editor.toSvg()`. Check which API is available.
Safer approach: use `editor.getSvgElement(editor.getCurrentPageShapeIds())` then convert to PNG via canvas.
Store editor ref in ClassroomWhiteboard and expose a `captureScreenshot(): Promise<string>` method via ref/callback.

## Constraints
- GPT-4o only for vision (not Claude — no vision API on this stack)
- OpenAI SDK already installed (used for TTS)
- Non-fatal if eval fails — lesson can continue
- Screenshot should only capture the student drawing area, not teacher shapes (filter by shape ID prefix if possible, or just capture full board)
- Keep the prompt tight — elementary tutor evaluating one specific step
