# Plan 01-05: Full Loop Wiring

## Goal
Wire the complete lesson loop end-to-end so one full session flows without manual intervention:
**Start → Generate → Teach (TTS) → Student draws → Evaluate (vision) → Feedback (TTS) → Next problem → ... → Complete**

Right now the pieces exist but aren't fully connected:
- Phase transitions are incomplete (evaluation result doesn't auto-advance)
- TTS isn't called for evaluation feedback
- "lesson complete" state has no UI
- `captureScreenshot` in ClassroomV3 uses `result.url` from `toImageDataUrl` — verify this actually returns a data URL (not a blob URL that won't work server-side). If it doesn't, switch to canvas-based capture.

## What To Build

### 1. Fix phase state machine in ClassroomV3.tsx
Current phases: `idle → teaching → student_turn → evaluating → reviewing`

Wire the full flow:
```
idle
  → (startLesson) → generating
  → (lesson ready, render first problem) → teaching
  → (TTS done) → student_turn
  → (checkWork) → evaluating
  → (result arrives) → reviewing
    → if correct + more problems: → teaching (next problem)
    → if correct + last problem: → complete
    → if incorrect: → student_turn (with hint visible, try again)
  → complete → idle (with "Start New Lesson" button)
```

Add `'generating'` and `'complete'` to `ClassroomPhase` in types.ts.

### 2. Auto-speak evaluation feedback
After `evaluateStudentWork` returns, call `speakText(evaluation.feedback)`.
If incorrect, also speak the hint: `speakText(evaluation.hint)` with a short pause.

### 3. Auto-advance on correct answer
When `evaluation.correct === true`:
- Speak feedback
- After TTS completes (or 2s delay if TTS doesn't return a promise), call `handleNextProblem()`
- If last problem: set phase to `'complete'`, speak a completion message

### 4. Retry on wrong answer
When `evaluation.correct === false`:
- Speak feedback + hint
- Stay in `student_turn` — don't advance
- Clear the student layer so they can try again (call `clearStudentLayer(editor)`)
- Keep hint visible in UI

### 5. Complete state UI in LessonControls.tsx
```
{phase === 'complete' && (
  <div className="text-center">
    <p className="text-2xl font-bold text-green-700">🎉 Lesson Complete!</p>
    <p className="text-gray-600 mt-2">Great work today!</p>
    <button onClick={onStartLesson} ...>Start New Lesson</button>
  </div>
)}
```

### 6. Screenshot verification
In ClassroomV3.tsx, check that `captureScreenshot` returns a proper data URL.
`editor.toImageDataUrl()` should return `{ url: 'data:image/png;base64,...' }`.
If `result.url` starts with `blob:` or is empty, switch to this approach:
```ts
const svgResult = await editor.getSvgElement(shapeIds)
// ... canvas render to data URL
```
Test by logging the first 30 chars of the screenshot before sending to API.
