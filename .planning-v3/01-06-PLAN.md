# Plan 01-06: Session Persistence

## Overview
Save classroom sessions and events to the database so we have a record of every lesson.
Use the EXISTING SQLite db (src/lib/db.ts) — do NOT add Supabase or new dependencies.

## Context
- Database: SQLite via better-sqlite3, already set up at src/lib/db.ts
- Do NOT use Supabase — check what db.ts already provides
- Keep it simple: two new tables, two API routes, wire into ClassroomV3.tsx

## Implementation

### Step 1: Check existing db.ts schema
Read src/lib/db.ts first. Understand what tables already exist. Add only what's missing.

### Step 2: Add tables (src/lib/classroom-v3/session-store.ts)
Create two functions:
- `createSession(topic: string, gradeLevel: number): string` — inserts a row into classroom_v3_sessions, returns sessionId (uuid)
- `logEvent(sessionId: string, type: string, data: object): void` — inserts a row into classroom_v3_events

Add these CREATE TABLE IF NOT EXISTS statements to run on import:
```sql
CREATE TABLE IF NOT EXISTS classroom_v3_sessions (
  id TEXT PRIMARY KEY,
  topic TEXT,
  grade_level INTEGER,
  started_at TEXT,
  ended_at TEXT
);
CREATE TABLE IF NOT EXISTS classroom_v3_events (
  id TEXT PRIMARY KEY,
  session_id TEXT,
  type TEXT,
  data TEXT,
  created_at TEXT
);
```

### Step 3: API routes
- POST /api/classroom/session — body: { topic, gradeLevel } → calls createSession, returns { sessionId }
- POST /api/classroom/event — body: { sessionId, type, data } → calls logEvent, returns { ok: true }

### Step 4: Wire into ClassroomV3.tsx
At lesson start: call /api/classroom/session, store sessionId in state
Log events for:
- lesson_started (topic, gradeLevel, lessonPlan)
- problem_shown (problemIndex, problemText)
- student_submitted (problemIndex, screenshot as base64 string if available, or empty string)
- evaluation_result (problemIndex, correct, feedback)
- lesson_complete

## Verification
1. npm run build passes
2. Start a lesson in /classroom-v3 and check that rows appear in the DB:
   `sqlite3 data/classroom.db "SELECT * FROM classroom_v3_sessions LIMIT 3;"`
