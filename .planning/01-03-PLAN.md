# 01-03 Plan — Thin V2 integration seam

Owner: Maximus
Executor: Rex
Status: Ready
Created: 2026-04-04 02:49 PT

## Goal
Create the first isolated app-facing integration seam for Classroom V2 so the new engine can exist inside the product without rewriting the live classroom loop.

This phase should make V2 tangible in the app while staying narrow and reversible.

## Scope
Build a thin, guarded integration path that:
1. mounts a small V2-aware classroom shell or runtime hook behind an explicit feature flag or isolated component boundary
2. instantiates the existing `classroom-v2` orchestrator with demo/placeholder adapters
3. surfaces derived V2 state in a reviewable way (state, allowed inputs, active turn / next events)
4. does **not** replace the current live tutoring flow yet

## Preferred Shape
One of these is acceptable:
- a `ClassroomV2Preview` / `ClassroomSessionV2` component gated by a feature flag
- a parallel route-level seam for review/demo use
- a small provider/hook + isolated component boundary consumed only when the flag is enabled

The seam should prove:
- the app can host the V2 engine
- the V2 orchestrator can drive UI-facing state
- future voice/board work can attach here instead of deepening legacy transcript-first code

## Implementation Requirements
1. Add a clearly named feature flag or review-only boundary for V2.
2. Keep legacy `VoiceSession` as default behavior unless the new seam is explicitly enabled.
3. Instantiate `createClassroomOrchestrator(...)` from the V2 module.
4. Render a minimal review surface showing at least:
   - current classroom state
   - allowed student input
   - active turn summary
   - allowed next event types
5. Add a tiny, explicit interaction path to advance the orchestrator through a demo lesson flow without vendor audio/browser dependencies.
6. Keep adapters placeholder-level only; no transcript parsing, no realtime/VAD expansion.
7. Update `.planning/01-03-RESULT.md` with what changed and how it was verified.

## Must-Haves
### Truths
- V2 now has a visible, app-level seam.
- The seam is isolated enough that future V2 work can land there instead of inside transcript-driven legacy code.
- The user can review V2 state progression without needing live audio.
- The repo still builds cleanly.

### Out of Scope
- full teacher layer / student layer whiteboard rebuild
- replacing the existing classroom production path
- realtime session rewiring
- transcript/logging migration
- final kid-facing UX polish

## Suggested Files To Touch
- new V2 review/seam component(s) under `src/components/` or `src/features/classroom-v2/`
- possibly `src/app/classroom/[studentId]/page.tsx` for boundary selection only
- optionally a small feature-flag helper under `src/lib/` or `src/config/`
- `.planning/01-03-RESULT.md`

## Acceptance Checks
- Another engineer can open the app and see a V2 review seam that is obviously separate from the legacy runtime.
- The seam exercises the real orchestrator rather than duplicating state in UI.
- Legacy behavior remains the default unless V2 is explicitly requested.
- `npm run build` passes.
