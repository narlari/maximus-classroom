# 06-11-PLAN — Targeted beat confidence signals inside preview

## Goal
Add one tiny preview-only confidence aid that exposes the exact live signals used for the targeted-beat verdict, so Sung can trust the verdict at a glance without interpreting hidden state.

## Why this slice
06-10 added a plain-language verdict, but confidence still depends on trusting that text. Showing the exact signal checks (live-target match + checklist confirmation) makes the verdict auditable in-place while keeping scope tiny.

## Scope
- Keep legacy `VoiceSession` as the default classroom runtime.
- Keep everything browser-local and review-only.
- Stay local to `ClassroomV2Preview` targeted-beat cue logic/UI.
- Extend the existing targeted cue with a compact `Confidence signals` strip that shows:
  - whether the requested beat is currently active on screen
  - whether the requested beat is marked done in the guided checklist
- Reuse existing launch-handoff, active-beat, and checklist state only.
- Update planning/state docs.

## Non-goals
- No backend/schema/database changes.
- No persistence model changes.
- No legacy runtime behavior changes.
- No broad preview redesign.

## Validation
- Launch targeted preview flows.
- Confirm the targeted cue now shows two explicit confidence signals and they change with preview state.
- Confirm verdict + signals remain consistent with checklist and active beat.
- Confirm non-targeted launches are unchanged.
- `npm run build` passes.
