# 06-03-PLAN — Guided V2 adoption checklist on the real classroom review path

## Goal
Add one narrow adoption slice that makes the real classroom route's V2 review path more concrete and reviewable by showing a small guided checklist that advances from actual V2 preview state.

## Why this slice
06-01 made the V2 seam reachable from the real classroom route. 06-02 made that route explain the handoff. The next highest-leverage step is to make the review path verifiable instead of just descriptive: once Sung launches the V2 seam from the real classroom route, he should be able to confirm the key adoption beats from a compact checklist driven by the live preview state.

## Scope
- Keep legacy `VoiceSession` as the default classroom experience.
- Keep all changes inside the route-shell / review seam; no legacy runtime cutover.
- Add a compact guided review checklist to `ClassroomV2Preview` that marks off the key V2 adoption beats as the demo advances:
  - teacher owns the setup
  - student turn opens clearly
  - answer handoff reaches teacher review
  - lesson lands intentionally
- If needed, add tiny local helpers to derive checklist completion from preview state/history.

## Non-goals
- No default classroom cutover
- No `VoiceSession` runtime behavior changes
- No schema/persistence/backend/orchestrator contract changes
- No broader preview redesign or more generic visual polish
- No new routing behavior

## Implementation sketch
1. Add a tiny review-checklist model inside `ClassroomV2Preview` based on current orchestrator state and recent history.
2. Render the checklist near the existing review controls so Sung can walk the seam from the real classroom route and see which V2 beats are already confirmed.
3. Keep the UI compact, explanatory, and reversible.
4. Update planning/state docs and verify with `npm run build`.

## Validation
- `/classroom/<studentId>` still opens legacy `VoiceSession` by default.
- Launching V2 preview from the real classroom route now exposes a live guided checklist tied to the preview state.
- The checklist confirms the core review beats without changing preview flow or legacy runtime behavior.
- `npm run build` passes.
