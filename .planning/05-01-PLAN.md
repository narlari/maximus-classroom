# 05-01-PLAN — Child-facing turn ownership + classroom cue polish in the V2 preview seam

## Goal
Make the isolated V2 preview feel more like a real classroom for a child reviewer by making turn ownership, current status, and immediate next step obvious, while keeping the legacy runtime, persisted schema, and parent-review flow untouched.

## Scope
- Stay primarily inside `src/features/classroom-v2/ClassroomV2Preview.tsx`.
- Keep all existing preview controls and debug/review surfaces available.
- Reorganize the preview so child-facing cueing is clearly separated from technical inspection areas.
- Keep the work compatibility-safe and review-only.

## Planned implementation
1. Add a child-readable “classroom now” layer near the top of the preview that clearly answers:
   - who owns the turn right now
   - what the student can do right now
   - what happens next
2. Add lightweight session-feel polish around current classroom state, active turn, and board section so the screen reads more like a lesson and less like raw instrumentation.
3. Preserve all existing controls and debug data, but move them into explicitly labeled review / technical inspection sections so reviewers can distinguish the kid-facing surface from the inspector surface.
4. Keep all logic additive and local to the preview seam, with no runtime default changes, schema changes, or broader orchestrator redesign.
5. Update `.planning/05-01-RESULT.md` and refresh `.planning/STATE.md` after implementation.
6. Run `npm run build` and finish only if it passes.

## Out of scope
- Any legacy `VoiceSession` default behavior changes
- DB or schema changes
- Parent dashboard expansion
- Broad classroom orchestrator API redesign
- Full animation system work beyond lightweight preview polish
