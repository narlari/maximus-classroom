# 04-02-PLAN — Expose full transcript and key events clearly in parent review

## Goal
Use the new `metadata.v2Event` seam from 04-01 to make parent/session review easier to read without changing the live classroom runtime.

## Scope
- Keep runtime behavior and persisted legacy event rows intact.
- Add a narrow review helper that interprets `SessionEvent.metadata.v2Event` into parent-friendly transcript and key-event artifacts.
- Update `ParentDashboardClient.tsx` so session review shows:
  - clearer tutor vs student transcript turns,
  - key events labeled/grouped by V2 state/action semantics when available,
  - board/system events surfaced as understandable review items instead of raw `event_type` dumps.

## Planned implementation
1. Add a small review helper under `src/lib/classroom-v2/` that:
   - reads `SessionEvent` rows,
   - extracts `metadata.v2Event` safely,
   - derives transcript entries with stable speaker labels,
   - derives key-event entries with readable titles/details/categories from V2 metadata,
   - falls back to legacy event info when V2 metadata is absent.
2. Update `ParentDashboardClient.tsx` to use the helper for session review rendering.
3. Keep styling changes narrow and local to the existing review cards.
4. Update `.planning/04-02-RESULT.md` and refresh `.planning/STATE.md` after implementation.
5. Run `npm run build` and only finish if it passes.

## Out of scope
- VoiceSession/runtime flow rewrites
- Progress/mastery hooks (04-03)
- Schema migrations
- Broad dashboard redesign
- V2 default runtime cutover
