# 06-01-PLAN — Explicit in-app V2 launcher on the real classroom page

## Goal
Add one narrow adoption slice that lets Sung reach the isolated V2 review seam through the app itself, without making V2 the default classroom path.

## Why this slice
The V2 seam is already reviewable, but today it is only reachable through a hidden query-parameter path. The next highest-leverage step is not more preview polish — it is a small, explicit opt-in entry inside the real classroom route so review can happen from the actual product path.

## Scope
- Keep legacy `VoiceSession` as the default classroom experience.
- Add a small in-app launcher on `src/app/classroom/[studentId]/page.tsx` when the page is on the legacy path.
- Add a clear return-to-default launcher when the page is in V2 preview mode.
- Keep the existing `?v2=...` flag behavior intact under the hood.
- If helpful, add tiny feature-flag URL helpers so the page logic stays narrow and readable.

## Non-goals
- No default classroom cutover
- No `VoiceSession` runtime behavior changes
- No schema/persistence/backend contract changes
- No broad routing refactor
- No further preview-only cosmetic polish inside `ClassroomV2Preview`

## Implementation sketch
1. Extend the V2 feature-flag helper with a tiny URL helper for enabling/disabling the preview query param.
2. Update `src/app/classroom/[studentId]/page.tsx` to render:
   - legacy classroom by default
   - a visible opt-in V2 preview launcher when on legacy
   - a visible return-to-default link when in preview
3. Keep the UI small, obvious, and review-safe so it functions as adoption plumbing rather than product redesign.
4. Update planning/state docs and verify with `npm run build`.

## Validation
- Opening `/classroom/<studentId>` still shows legacy `VoiceSession` by default.
- Sung can reach the V2 seam from inside the app without manually typing `?v2=1`.
- V2 preview still remains opt-in only.
- `npm run build` must pass.
