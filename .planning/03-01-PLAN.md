# 03-01-PLAN — Wrap existing realtime transport behind the V2 orchestration adapter

## Goal
Create a narrow V2-facing seam where tutor speech and student mic policy can be driven through orchestration-facing interfaces instead of direct ad hoc coupling to the realtime transport.

## Scope
- Add a V2 realtime adapter module under `src/lib/classroom-v2/`
- Define a small transport interface that V2 can target without depending on browser/WebRTC details
- Wrap the existing browser realtime session helper behind that transport shape
- Expose the seam in the isolated V2 preview with an in-memory transport so reviewers can inspect orchestration-driven transport commands without touching live runtime behavior
- Update planning/state docs for this slice

## Explicitly Out of Scope
- No 03-02 mic/session enforcement beyond adapter-level mic mute/open signaling
- No 03-03 student voice evidence normalization
- No whiteboard redesign
- No default-runtime cutover of `VoiceSession`

## Intended Code Shape
1. `classroom-v2` owns a transport-agnostic adapter that reacts to orchestrator snapshots.
2. Speech actions (`speak` / `ask`) are translated into transport `speak(...)` calls by the adapter.
3. Student input state changes are translated into transport mic mute policy by the adapter.
4. Existing realtime browser/WebRTC code remains usable, but behind a transport wrapper instead of being the only control model.
5. The V2 preview shows the seam with a memory transport so the change is reviewable without live audio.

## Success Criteria
- There is a reviewable adapter seam between V2 orchestration and realtime transport
- Existing default classroom behavior remains unchanged unless a V2 seam explicitly uses the adapter
- The code makes it obvious that V2 targets a transport interface, not raw realtime calls
- `npm run build` passes
