# 03-04-PLAN — Route real browser student transcripts into the V2 voice-evidence seam

## Goal
Capture real browser realtime student transcript events and pass them through the existing V2 voice-evidence packaging helpers in a narrow, reviewable way, without forcing a broader classroom runtime cutover.

## Scope
- Add a small V2 transcript adapter/helper for browser realtime student transcript events
- Extend the browser realtime surface just enough to expose a structured student transcript event to callers
- Wire legacy `VoiceSession` student transcript handling through that adapter so real transcript events produce V2-packaged voice evidence metadata
- Keep legacy classroom behavior stable and default-first
- Update planning/state docs and run `npm run build`

## Explicitly Out of Scope
- No parent dashboard work
- No major `VoiceSession` rewrite
- No full V2 orchestrator/runtime cutover
- No unrelated cleanup

## Intended Code Shape
1. Add a V2 helper that converts browser realtime student transcript events into `createVoiceSubmissionEvidence(...)` inputs.
2. Extend `src/lib/realtime.ts` so completed student transcript callbacks receive a small structured event object, not just raw text.
3. Update `VoiceSession` to keep using its legacy runtime flow, but attach V2-packaged voice evidence metadata when real student transcript events are logged.
4. Keep all new behavior isolated behind helper/adaptor code so future V2 runtime adoption can swap in richer turn context later.

## Success Criteria
- Real browser student transcript events reach a dedicated V2 adapter/helper seam
- That seam routes transcript data through the existing V2 voice-evidence packaging helpers
- Legacy `VoiceSession` behavior remains stable by default
- The slice is reviewable in isolation
- `npm run build` passes
