# 06-07-PLAN — Route-level targeted review beat launcher

## Goal
Let the real classroom route launch the V2 preview directly into one specific review beat — teacher setup, student turn, review handoff, or lesson close — so Sung can inspect that moment without replaying the whole preview from the top first.

## Why this slice
06-06 already carries remembered start / continue / re-check guidance into preview entry. The remaining friction is targeted inspection: even with remembered guidance, Sung still has to step through intermediate beats when he wants to sanity-check one exact review moment. This slice should remove that friction from the real classroom route while keeping the work review-only and browser-local.

## Scope
- Keep legacy `VoiceSession` as the default classroom runtime.
- Keep changes browser-local and review-only.
- Reuse the existing route-to-preview launch handoff / query-param path.
- Add small route-level controls that launch preview directly into each review beat.
- Make preview clearly acknowledge when it was launched for a targeted beat review.
- Preserve the existing remembered start / continue / re-check guidance and launcher behavior.
- Update planning/state docs.

## Non-goals
- No backend/schema/database changes.
- No default cutover to V2.
- No legacy `VoiceSession` behavior changes.
- No broad preview redesign.
- No new persistence model.

## Implementation sketch
1. Extend the browser-local review-memory helper with a targeted-launch handoff shape built on the existing review mode / beat query params.
2. Update the real classroom review card to keep the remembered launcher intact while also rendering explicit per-beat launch links.
3. Update preview entry parsing + launch banner copy so preview distinguishes between remembered resume/re-check launches and explicit targeted beat review launches.
4. Reuse the existing preview prepositioning logic so targeted route launches open directly at the requested beat.
5. Update `.planning/STATE.md` and add a concise result doc.
6. Validate with `npm run build`.

## Validation
- From `/classroom/<studentId>`, each direct beat launcher opens preview at the requested beat.
- The default remembered launcher still behaves exactly as before.
- Preview entry copy clearly says when it was opened for targeted beat review and names the requested beat.
- Legacy classroom route still defaults to `VoiceSession` unless preview is explicitly launched.
- `npm run build` passes.
