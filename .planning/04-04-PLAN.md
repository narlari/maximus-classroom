# 04-04-PLAN — Add review-only standards / evaluator hints from V2 event metadata

## Goal
Use the new 04-03 review-only progress seam to derive tentative standards/domain/evaluator hints from structured V2 session events so a parent reviewer can inspect session-by-session scoring context without changing persisted mastery data or default classroom runtime behavior.

## Scope
- Keep legacy runtime behavior as default.
- Keep persisted event rows and schema unchanged.
- Keep existing `skill_progress` and `applySkillUpdates` path unchanged.
- Add one narrow helper under `src/lib/classroom-v2/` that reads `SessionEvent.metadata.v2Event` plus existing review/progress context.
- Surface the derived hints in the parent dashboard only enough to be inspectable per session.

## Planned implementation
1. Add `src/lib/classroom-v2/progress-hints.ts` that:
   - safely reads `metadata.v2Event` and optional `metadata.v2VoiceEvidence`,
   - derives tentative domain tags from prompt/action/evidence text,
   - maps those hints against existing student progress standards when possible,
   - emits evaluator labels and review-only scoring inputs,
   - remains fully compatibility-safe when V2 metadata is absent.
2. Export the helper from `src/lib/classroom-v2/index.ts`.
3. Update `ParentDashboardClient.tsx` to show:
   - a compact student-level summary of recent evaluator hints,
   - per-session standards/evaluator hint cards under the existing progress hooks.
4. Refresh `.planning/04-04-RESULT.md` and `.planning/STATE.md`.
5. Run `npm run build` and only finish if it passes.

## Out of scope
- Runtime cutover to a V2 evaluator
- VoiceSession/orchestrator rewrite
- Schema migration
- Replacing legacy mastery scoring or `applySkillUpdates`
- Any claim of authoritative standards scoring from event streams
