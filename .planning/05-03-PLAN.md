# 05-03-PLAN — Lesson-transition smoothing in the isolated V2 preview seam

## Goal
Make the isolated V2 preview read like one continuous lesson by adding lightweight browser-local transition cues around key handoffs between teacher-owned and student-owned beats, while keeping the work narrow, review-only, and primarily local to `src/features/classroom-v2/ClassroomV2Preview.tsx`.

## Scope
- Stay primarily inside `src/features/classroom-v2/ClassroomV2Preview.tsx`.
- Add temporary, readable handoff cues for key transitions:
  - `tutor_rendering -> tutor_speaking`
  - `tutor_speaking -> student_answering`
  - `student_submitted -> tutor_reviewing`
  - `tutor_reviewing -> next tutor turn or lesson end`
- Improve continuity framing with simple “what just happened / what happens next” copy, handoff strips, or similar review-friendly preview cues.
- Preserve all existing technical inspection/debug surfaces.
- Keep behavior browser-local only with no backend/runtime/schema/orchestrator contract changes.

## Planned implementation
1. Add a preview-local transition memory model that notices recent state changes and derives a short-lived handoff snapshot for the current lesson beat.
2. Surface that snapshot in the child-facing preview as a compact transition strip/card that explains:
   - what just finished
   - what is starting now
   - what the student should expect next
3. Reuse the transition snapshot near the board/lesson-stage area so the board surface keeps continuity across handoffs instead of feeling like abrupt state swaps.
4. Keep all logic additive and local to the preview seam; do not change legacy runtime defaults, persistence, mastery logic, or parent/dashboard behavior.
5. Update `.planning/05-03-RESULT.md` and refresh `.planning/STATE.md` after implementation.
6. Run `npm run build` and only finish if it passes.

## Out of scope
- Legacy `VoiceSession` default changes
- Backend/runtime/schema/orchestrator contract changes
- Parent dashboard changes
- Persistence/mastery logic changes
- Flashy animation systems or broad UI restyling
