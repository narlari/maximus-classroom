# 05-05-PLAN — Subtle teacher-board motion / reveal refinement in the isolated V2 preview seam

## Goal
Make the isolated V2 preview’s teacher-owned board beats feel slightly more alive and intentionally paced by softly staging board elements as they appear, without introducing any runtime scheduling system, backend changes, or broad animation framework work.

## Scope
- Stay narrowly inside the isolated V2 preview seam and the canonical V2 board presentation layer.
- Reuse the existing deterministic preview pacing snapshot instead of creating a new timing system.
- Add subtle teacher-layer reveal motion for teacher-owned beats and calmer settled presentation for student/review beats.
- Preserve all existing debug / inspection surfaces and keep student drawing behavior intact.

## Planned implementation
1. Update `src/features/classroom-v2/ClassroomV2Preview.tsx` to pass a narrow teacher reveal snapshot into the canonical whiteboard boundary:
   - current reveal progress percent
   - active action index
   - whether the current beat is teacher-led / should animate
2. Update `src/components/classroom-whiteboard/CanonicalWhiteboard.tsx` to thread those preview-only reveal props through the canonical layered board seam.
3. Update `src/lib/classroom-v2/board.ts` so teacher board primitives retain their source action index, allowing deterministic per-action reveal staging without changing action contracts.
4. Update `src/components/classroom-whiteboard/LayeredBoard.tsx` so teacher-layer primitives use small browser-local opacity / translate / scale / stroke-draw transitions keyed off the passed reveal snapshot:
   - already revealed elements feel settled
   - the active teacher action gets a subtle reveal emphasis
   - queued elements stay hidden until their turn
   - non-teacher beats fall back to calm fully-shown rendering
5. Write `.planning/05-05-RESULT.md`, update `.planning/STATE.md`, and run `npm run build`.

## Out of scope
- Legacy classroom cutover
- VoiceSession default changes
- DB / persistence / backend / runtime contract changes
- Parent dashboard changes
- Broad animation infrastructure or real runtime scheduling
