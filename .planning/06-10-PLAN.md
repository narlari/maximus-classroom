# 06-10-PLAN — Targeted beat verdict strip inside preview

## Goal
Add one tiny preview-only confidence aid that tells Sung, in plain language, whether the requested targeted beat is still pending, open now, or already confirmed.

## Why this slice
06-09 explains what the targeted beat means, but Sung still has to infer whether the live preview has actually satisfied that checkpoint yet. A tiny verdict strip removes that last bit of interpretation.

## Scope
- Keep legacy `VoiceSession` as the default classroom runtime.
- Keep everything browser-local and review-only.
- Stay local to `ClassroomV2Preview` and existing targeted-beat helper logic.
- Extend the targeted beat cue with a compact live verdict strip that:
  - states whether the requested beat is pending, open now, or confirmed
  - states the live signal that produced that verdict
  - reuses existing launch-handoff, checklist, and active-beat state
- Update planning/state docs.

## Non-goals
- No backend/schema/database changes.
- No persistence model changes.
- No broader preview redesign.
- No changes to runtime orchestration behavior.

## Validation
- Launch preview into each targeted beat state that is easy to reach.
- Confirm the targeted cue now includes a plain-language verdict derived from live preview state.
- Confirm the verdict changes from open/pending to confirmed once preview crosses the requested checkpoint.
- Confirm non-targeted launches still behave the same.
- `npm run build` passes.
