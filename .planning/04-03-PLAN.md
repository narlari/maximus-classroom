# 04-03-PLAN — Add progress hooks tied to structured lesson events

## Goal
Use the V2 event metadata and clearer parent-review seam from 04-01/04-02 to derive narrow, reviewable progress/mastery signals from structured lesson/session events without changing the default classroom runtime or replacing the existing summary-driven mastery updates.

## Scope
- Keep legacy runtime behavior as default.
- Keep persisted session rows and existing `skill_progress` schema unchanged.
- Add a small helper under `src/lib/classroom-v2/` that interprets `SessionEvent.metadata.v2Event` into progress-relevant review signals.
- Thread those signals into the parent-facing review/progress path only enough to make the slice inspectable in the dashboard.

## Planned implementation
1. Add a dedicated V2 progress helper that:
   - safely reads `metadata.v2Event`,
   - derives progress-relevant session signals from tutor/student/board/system events,
   - summarizes counts for practice attempts, voice responses, board submissions, and review moments,
   - falls back safely when V2 metadata is absent.
2. Export the helper from `src/lib/classroom-v2/index.ts`.
3. Update `ParentDashboardClient.tsx` to surface:
   - a concise recent structured-signal summary in the progress area,
   - per-session progress signals/details alongside the existing transcript and key-event review.
4. Update `.planning/04-03-RESULT.md` and refresh `.planning/STATE.md` after implementation.
5. Run `npm run build` and only finish if it passes.

## Out of scope
- Broad runtime cutover
- VoiceSession/orchestrator rewrites
- Schema migrations
- Replacing summary/LLM-driven `applySkillUpdates`
- Full mastery scoring from event streams
