# 04-06-PLAN — Clearer evaluator rubric + parent explanation signals in the review-only hint seam

## Goal
Add a small, explicit rubric/explanation layer to the existing Phase 4 parent-review hints so parents can more clearly see why a voice / board / combined card is reviewable, without changing legacy runtime defaults, persisted schema, or summary-driven mastery.

## Scope
- Stay inside the existing `progress-hints` + parent dashboard review seam.
- Keep all runtime behavior unchanged by default.
- Keep persistence/schema unchanged.
- Keep `skill_progress`, `applySkillUpdates`, and summary-driven mastery untouched.
- Remain compatibility-safe when optional V2 metadata is missing.

## Planned implementation
1. Extend `src/lib/classroom-v2/progress-hints.ts` with additive rubric/explanation helpers that derive review-only metadata from current event / evidence context, including:
   - response modality (voice / board / voice + board)
   - evidence completeness / context richness
   - strategy marker presence
   - prompt-linked evidence presence
   - short parent-facing explanation text
2. Thread those rubric/explanation fields through `ParentProgressHintCard` and `ParentProgressHintSummary` without changing existing hint consumers.
3. Update `src/components/ParentDashboardClient.tsx` just enough to surface the new rubric/explanation outputs on review cards and lightweight summary chips, without redesigning the page.
4. Refresh `.planning/04-06-RESULT.md` and `.planning/STATE.md` to record the slice and reaffirm review-only boundaries.
5. Run `npm run build` and finish only if it passes.

## Out of scope
- Any classroom runtime or orchestrator cutover
- DB migrations or persisted shape changes
- Replacing legacy mastery / progress persistence
- UI redesign beyond small additive parent-review rendering
