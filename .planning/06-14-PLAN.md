# 06-14-PLAN — Targeted beat return-vs-replay guidance inside preview

## Goal
Add one more tiny preview-only aid to the targeted-beat cue so Sung can see plainly whether this run is safe to leave and return to the real classroom route, or whether a replay is still the recommended next move.

## Why this slice
06-13 explained what review progress looks like after the requested checkpoint, but Sung still has to translate that into an exit decision. A compact return-guidance panel keeps the same targeted-review seam, stays browser-local, and removes one more interpretation step without widening scope.

## Scope
- Keep legacy `VoiceSession` as the default classroom runtime.
- Keep everything browser-local and review-only.
- Stay local to `ClassroomV2Preview` targeted-beat cue logic/UI.
- Extend the existing targeted cue with a compact `Return to classroom route?` panel derived only from:
  - requested targeted beat
  - current checklist completion state
  - whether the requested beat is active now
  - whether another review beat remains unresolved when helpful
- Keep existing verdict, confidence signals, recommended action, projected progress, confirmation, and follow-up content intact.
- Update planning/state docs.

## Non-goals
- No backend/schema/database changes.
- No persistence model changes.
- No legacy runtime behavior changes.
- No route-level redesign.

## Validation
- Launch targeted preview flows.
- Confirm the targeted cue now states whether it is safe to return to the classroom route or whether replay/continue is still recommended.
- Confirm the return guidance reads correctly for already-confirmed, open-now, and advanced/not-yet-confirmed targeted launches.
- Confirm non-targeted launches are unchanged.
- `npm run build` passes.
