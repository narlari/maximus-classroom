# 05-07-PLAN — Lightweight teacher-review handoff emphasis in the isolated V2 preview seam

## Goal
Add one narrow Phase 5 polish slice that makes the student-submission → teacher-review handoff read more intentionally in the isolated V2 preview seam, without changing runtime behavior or contracts.

## Why this slice
05-05 and 05-06 already improved teacher reveal pacing and end-of-beat settling. The next tight follow-up is to make the review handoff feel visibly acknowledged on the child-facing board surface so the moment between "answer received" and "Maximus is reviewing" feels teacher-led instead of merely stateful.

## Scope
- Stay browser-local and preview-only.
- Reuse the existing isolated preview seam and canonical whiteboard presentation layer.
- Add a small review-handoff snapshot derived from preview state/timing.
- Surface that snapshot in:
  - `ClassroomV2Preview.tsx` child-facing review copy
  - `CanonicalWhiteboard.tsx` prop passthrough
  - `LayeredBoard.tsx` board-local handoff banner / framing cue

## Non-goals
- No legacy runtime default changes
- No `VoiceSession` behavior changes
- No schema/persistence/data-flow changes
- No parent dashboard or backend/orchestrator contract changes
- No new whiteboard seam

## Implementation sketch
1. Derive a narrow `ReviewHandoffSnapshot` from preview state/timing.
2. Add a small review-handoff cue card in the preview lesson panel.
3. Pass preview-only handoff props through the canonical board boundary.
4. Render a subtle board-local handoff banner and frame emphasis during `student_submitted` and the early `tutor_reviewing` beat.
5. Update result/state docs and verify with `npm run build`.

## Validation
- `student_submitted` should read like "answer received, handing back to Maximus".
- Early `tutor_reviewing` should read like "Maximus is now inspecting the answer".
- Student tools/runtime behavior should remain unchanged.
- `npm run build` must pass.
