# 06-04-PLAN — Route-level V2 review memory

## Goal
Add one narrow adoption slice that lets the real classroom route remember the latest confirmed V2 review beats per student and replay that memory around the existing launcher / return controls.

## Why this slice
06-01 made the V2 seam reachable from the real classroom route. 06-02 explained the handoff there. 06-03 made the launched seam verifiable with a live checklist. The next highest-leverage step is to preserve that checklist progress in a tiny browser-local way so Sung can leave and re-enter `/classroom/<studentId>` without losing the last confirmed V2 review beats.

## Scope
- Keep legacy `VoiceSession` as the default classroom experience.
- Keep changes narrow and local to the route-shell / preview seam.
- Add tiny browser-local review memory keyed per student.
- Persist the V2 preview checklist progress into that memory.
- Surface the remembered beats near the existing route-level launcher / return controls.
- Keep reset behavior obvious by separating demo reset from remembered-review reset.

## Non-goals
- No default classroom cutover.
- No `VoiceSession` runtime behavior changes.
- No backend/schema/persistence changes outside browser-local storage.
- No orchestrator contract changes.
- No broad preview redesign.

## Implementation sketch
1. Add a tiny `review-memory` helper in `src/lib/classroom-v2/` that reads/writes/clears per-student browser-local records.
2. Persist the existing guided checklist progress from `ClassroomV2Preview` into that helper.
3. Replace the real classroom route's static launcher card with a small client card that can replay the remembered beats near the existing launcher/return controls.
4. Make reset behavior explicit:
   - `Reset demo only` keeps remembered route-level review memory.
   - `Reset demo + clear remembered review` wipes both the preview run and the stored route memory.
5. Update planning/state docs and verify with `npm run build`.

## Validation
- `/classroom/<studentId>` still opens legacy `VoiceSession` by default.
- Launching V2 preview still works.
- Guided checklist progress is remembered per student in lightweight browser-local storage.
- Returning to the real classroom route replays the last confirmed V2 review beats near the launcher.
- Reset behavior is obvious and sensible.
- `npm run build` passes.
