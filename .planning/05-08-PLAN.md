# 05-08-PLAN — Tiny lesson-close landing cues in the isolated V2 preview seam

## Goal
Add one narrow Phase 5 polish slice that makes the post-review / lesson-close beat feel intentionally finished in the isolated V2 preview seam, without changing runtime behavior or contracts.

## Why this slice
05-05 through 05-07 already improved reveal motion, settled holds, and the student-submission → teacher-review handoff. The next highest-leverage micro-slice is the final landing cue: when review resolves into lesson end, the preview should read like Maximus intentionally closed the lesson instead of abruptly switching to an ended state.

## Scope
- Stay browser-local and preview-only.
- Reuse the existing isolated preview seam and canonical whiteboard presentation layer.
- Add a small lesson-close snapshot derived from preview state/timing.
- Surface that snapshot in:
  - `ClassroomV2Preview.tsx` child-facing close copy
  - `CanonicalWhiteboard.tsx` prop passthrough
  - `LayeredBoard.tsx` board-local lesson-landed cue

## Non-goals
- No legacy runtime default changes
- No `VoiceSession` behavior changes
- No schema/persistence/data-flow changes
- No parent dashboard or backend/orchestrator contract changes
- No new whiteboard seam
- No broad refactors

## Implementation sketch
1. Derive a narrow `LessonCloseSnapshot` from preview state/timing.
2. Add a small lesson-close cue card in the preview lesson panel so the final beat reads like a deliberate finish.
3. Pass preview-only lesson-close props through the canonical board boundary.
4. Render a subtle board-local lesson-landed banner / progress cue during the ended landing window.
5. Update result/state docs and verify with `npm run build`.

## Validation
- Early `ended` should read like the lesson just landed, not like a generic stopped state.
- The board should carry a subtle finality cue in the same moment.
- Student tools/runtime behavior should remain unchanged.
- `npm run build` must pass.
