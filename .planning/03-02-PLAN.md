# 03-02-PLAN — Enforce state-aware mic/session rules

## Goal
Make the V2 realtime adapter enforce the classroom state machine's mic/session ownership rules from one obvious policy seam under `src/lib/classroom-v2/`, without cutting over the default classroom runtime.

## Scope
- Add a dedicated V2 realtime session policy module under `src/lib/classroom-v2/`
- Define the effective student voice-capture policy by classroom state
- Route V2 realtime adapter mic/session behavior through that policy instead of ad hoc `studentInput.mic` toggles
- Let the browser realtime transport consume the policy through a transport-shaped session-control hook
- Expose the effective policy in the V2 preview so reviewers can verify tutor-turn enforcement
- Update planning/state docs and run `npm run build`

## Explicitly Out of Scope
- No 03-03 student voice evidence normalization
- No legacy `VoiceSession` cutover or broad runtime refactor
- No parent dashboard work
- No whiteboard redesign

## Intended Code Shape
1. `src/lib/classroom-v2/realtime-policy.ts` becomes the single obvious place that maps V2 classroom state to effective mic/session behavior.
2. `createClassroomRealtimeOrchestrationAdapter(...)` applies that policy whenever state ownership changes.
3. Browser/in-memory transports expose `applySessionPolicy(...)` so the adapter talks in V2 policy terms instead of scattered mic toggles.
4. Tutor-owned states (`tutor_rendering`, `tutor_speaking`, `student_submitted`, `tutor_reviewing`) keep student voice capture closed/ignored.
5. `student_answering` is the only state that opens active student voice capture.

## Success Criteria
- There is one obvious V2 policy seam for mic/session rules
- Tutor-owned states visibly lock student voice capture in the preview
- `student_answering` is the only state that opens voice capture
- Legacy default behavior remains unchanged unless a V2 seam opts into the new policy
- `npm run build` passes
