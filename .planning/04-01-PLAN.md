# 04-01-PLAN — Align event logging with V2 states/actions

## Goal
Make existing runtime session logs carry clearer V2 classroom state/action semantics through a narrow helper seam, without changing the default legacy classroom behavior or attempting a broader runtime cutover.

## Scope
- Add a dedicated V2 event-logging helper under `src/lib/classroom-v2/`
- Map the current legacy `VoiceSession` logging path into V2-aligned state/action metadata for key runtime events
- Keep the existing session event types and legacy flow intact while enriching metadata for reviewability
- Update planning/state docs and run `npm run build`

## Explicitly Out of Scope
- No parent dashboard UI work
- No broad `VoiceSession` rewrite
- No full V2 orchestrator/session runtime cutover
- No 04-02 work
- No unrelated cleanup

## Intended Code Shape
1. Add a small helper that converts legacy runtime events into typed V2 event semantics metadata.
2. Use that helper from `VoiceSession` for the most important existing event artifacts:
   - student voice exchange
   - tutor voice exchange
   - whiteboard snapshot
   - system notice
3. Keep existing event rows and behavior stable, but make metadata explicitly describe V2 state/action meaning so later parent/review work has a clearer substrate.

## Success Criteria
- A reviewer can inspect one obvious helper seam that maps legacy runtime events to V2 state/action metadata.
- Existing `voice_exchange`, `whiteboard_snapshot`, and `system_notice` rows keep logging as before.
- Those rows now include clearer V2-aligned semantics or evidence metadata.
- Legacy default classroom behavior remains intact.
- `npm run build` passes.
