# 02-03 Plan — Submit/check snapshots and evidence packaging

Owner: Maximus
Executor: Rex
Status: In progress
Started: 2026-04-04

## Goal
Wire a narrow, reviewable submit/check flow inside the existing V2 preview seam so explicit student submissions capture the isolated student layer and package the active tutor-turn context needed for review.

## Scope
- Stay inside the existing V2 preview/canonical whiteboard seam
- Preserve legacy default classroom behavior
- Do not begin Phase 3 voice integration
- Do not redesign the whiteboard architecture again

## Planned slice
1. Extend V2 evidence types with a review-oriented package shape for board submissions
   - include explicit student-layer snapshot metadata
   - include teacher/tutor review context from the active turn
2. Add a small packaging helper in `src/lib/classroom-v2/`
   - build the evidence object intentionally from board capture + active turn context
3. Update `LayeredBoard`
   - retain student strokes in state while drawing
   - expose explicit `Submit work` / `Check my work` actions in V2 mode
   - capture the isolated student layer as a snapshot on submit/check
4. Update the V2 preview seam
   - consume board submit/check callbacks instead of demo-only evidence submission
   - show review detail so a reviewer can inspect the packaged artifact shape
5. Build-verify with `npm run build`

## Acceptance checks
- `/classroom/<studentId>?v2=1` allows explicit student submit/check within the V2 seam
- submission packages student-layer evidence with relevant active tutor-turn context
- reviewer can see enough detail in the seam to confirm the evidence package shape intentionally exists
- `npm run build` passes
