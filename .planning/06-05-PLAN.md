# 06-05-PLAN — Route-level V2 review resume guidance

## Goal
Add one narrow adoption slice that makes the real classroom route's V2 review card smarter about resuming review from remembered checklist state.

## Why this slice
06-04 made route-level V2 review memory visible, but the card is still mostly a static launcher with a remembered list underneath it. The next narrow step is to turn that remembered state into actionable route guidance so Sung can immediately tell whether he should start, continue, or re-check the V2 review seam.

## Scope
- Keep legacy `VoiceSession` as the default classroom experience.
- Keep changes local to the route review card and browser-local review memory seam.
- Derive route-level review guidance from the remembered checklist state.
- Show whether the route should frame the next action as:
  - starting V2 review
  - continuing V2 review
  - re-checking / revisiting the V2 review seam
- Highlight the next unconfirmed review beat whenever review is in progress.
- Preserve the existing launcher / return behavior.

## Non-goals
- No default classroom cutover.
- No backend/schema/orchestrator/runtime changes.
- No changes to legacy `VoiceSession` behavior.
- No broad UI redesign.

## Implementation sketch
1. Add a tiny review-memory guidance helper that derives route copy from remembered checklist state.
2. Update the route-level `ClassroomV2ReviewMemoryCard` to use that helper for its main headline/body/summary.
3. Show the next unconfirmed review beat when only part of the checklist is confirmed.
4. When all beats are confirmed, shift the card framing to re-check / revisit language and summarize the route-level pass as complete.
5. Update planning/state docs and verify with `npm run build`.

## Validation
- With no remembered review memory, the card frames review as starting V2 review.
- With partial remembered review memory, the card frames review as continuing and highlights the next unconfirmed beat.
- With fully confirmed remembered review memory, the card frames review as re-checking / revisiting and summarizes completion.
- Launcher / return controls still behave exactly as before.
- Legacy classroom remains the default runtime.
- `npm run build` passes.
