# 06-12-PLAN — Targeted beat recommended next action inside preview

## Goal
Add one more tiny preview-only interpretation aid to the targeted-beat cue so Sung can tell the right immediate action at a glance, not just the verdict and underlying signals.

## Why this slice
06-11 made the verdict auditable, but Sung still has to translate that status into what to do next. A small state-driven recommended action keeps the same seam, reduces interpretation further, and stays browser-local.

## Scope
- Keep legacy `VoiceSession` as the default classroom runtime.
- Keep everything browser-local and review-only.
- Stay local to `ClassroomV2Preview` targeted-beat cue logic/UI.
- Extend the existing targeted cue with a compact `Recommended next action` panel derived only from:
  - requested targeted beat
  - whether that beat is active now
  - whether that beat is checklist-confirmed in this run
  - current active beat label when useful
- Keep existing verdict/signals/confirm/follow-up content intact.
- Update planning/state docs.

## Non-goals
- No backend/schema/database changes.
- No persistence model changes.
- No legacy runtime behavior changes.
- No broad preview redesign.

## Validation
- Launch targeted preview flows.
- Confirm the targeted cue now names the immediate next action for open, confirmed, and advanced/not-yet-confirmed states.
- Confirm the recommendation stays consistent with verdict + confidence signals.
- Confirm non-targeted launches are unchanged.
- `npm run build` passes.
