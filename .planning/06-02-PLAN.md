# 06-02-PLAN — Route-level review handoff around the in-app V2 launcher

## Goal
Make the V2 review path self-explanatory from the real classroom route itself now that 06-01 exposed the in-app launcher.

## Why this slice
06-01 solved discoverability by adding an explicit launcher into the V2 preview seam from the actual classroom page. The next narrow gap is review handoff clarity: once Sung lands on the real classroom route, the page should briefly explain how to review the seam and what to look for without relying on hidden/manual URL knowledge or separate planning docs.

## Scope
- Keep legacy `VoiceSession` as the default classroom experience.
- Keep the existing `?v2=1` preview toggle behavior intact.
- Add a very small route-shell review handoff panel on `src/app/classroom/[studentId]/page.tsx`:
  - on legacy mode: explain the 2–3 step review path before launching preview
  - on preview mode: explain what Sung is currently looking at and how to return to default
- Keep all changes local to the route shell UI; do not change `VoiceSession` runtime or V2 preview internals.

## Non-goals
- No default classroom cutover
- No `VoiceSession` behavior changes
- No preview-seam feature expansion or deeper polish work
- No schema/persistence/backend/orchestrator changes
- No routing refactor

## Implementation sketch
1. Add tiny route-shell copy helpers/constants for the review handoff steps and review focus points.
2. Update `src/app/classroom/[studentId]/page.tsx` so the top-right launcher area becomes a slightly clearer review handoff package:
   - legacy: small "Review path" card + launch CTA
   - preview: compact "Reviewing now" card + return CTA
3. Keep the visual footprint small and reversible.
4. Update planning/state docs and verify with `npm run build`.

## Validation
- `/classroom/<studentId>` still opens the legacy classroom by default.
- The launcher remains the only path into V2 preview from the real classroom route.
- The route itself now tells Sung how to review the seam and what he is looking at.
- `npm run build` passes.
