# 04-05-PLAN — Richer board / combined-evidence scoring inputs in the review-only parent seam

## Goal
Package richer evaluator-ready board context into the existing Phase 4 parent-review seam and let same-turn voice + board evidence collapse into a clearer combined review card, without changing legacy runtime defaults, persisted schema, or summary-driven mastery.

## Scope
- Keep legacy runtime behavior as default.
- Keep persisted event rows/schema unchanged.
- Keep existing `skill_progress` and `applySkillUpdates` behavior unchanged.
- Stay inside the existing review-only seam already used by parent review / progress hints.
- Remain compatibility-safe when V2 metadata is absent.

## Planned implementation
1. Extend the legacy whiteboard event metadata helper so whiteboard snapshot rows can carry a narrow `v2BoardEvidence` payload:
   - source
   - captured timestamp
   - board description summary
   - packaged review context / teacher actions for the active turn
2. Update `VoiceSession` whiteboard snapshot logging to pass the description into that metadata helper.
3. Upgrade `src/lib/classroom-v2/progress-hints.ts` so review cards can:
   - read optional board metadata alongside existing voice metadata,
   - use board summaries + teacher actions as richer scoring inputs,
   - collapse same-turn voice and board rows into a combined evaluator-ready hint,
   - expose board context labels/summary in a review-only way.
4. Update `ParentDashboardClient` just enough to surface:
   - combined voice+board counts in the student summary,
   - board context details on per-session evaluator hint cards.
5. Refresh `.planning/04-05-RESULT.md` and `.planning/STATE.md`.
6. Run `npm run build` and only finish if it passes.

## Out of scope
- VoiceSession rewrite
- Orchestrator/runtime cutover
- DB migration
- Replacing mastery scoring or `applySkillUpdates`
- Phase 5 polish or classroom UI redesign
