# 06-06-PLAN — Route-to-preview review resume handoff

## Goal
Turn the existing route-level V2 review resume guidance into an actual preview launch handoff so the V2 preview opens already framed around the remembered review mode and resume point.

## Why this slice
06-05 taught the real classroom route how to explain whether Sung should start, continue, or re-check the review seam, but that guidance still stops at the launcher. Once preview opens, Sung still has to rediscover where the remembered pass left off. The next narrow step is to carry that route-level guidance into preview entry itself.

## Scope
- Keep legacy `VoiceSession` as the default classroom runtime.
- Keep changes browser-local and review-only.
- Add a tiny route-to-preview handoff payload derived from remembered review memory.
- Make the launched preview acknowledge whether the remembered pass is a start / continue / re-check flow.
- Highlight the remembered next unconfirmed review beat immediately on preview entry.
- When resuming a partial pass, initialize the preview at or near the next unconfirmed beat so the relevant state is visible right away.
- When all beats are complete, frame preview as a re-check / confidence pass without changing checklist semantics.
- Preserve existing launch / return behavior.

## Non-goals
- No backend/schema/database changes.
- No default cutover to V2.
- No legacy `VoiceSession` behavior changes.
- No broad preview redesign.
- No changes to checklist rules or completion logic.

## Implementation sketch
1. Extend the review-memory helper with a tiny serializable launch handoff shape derived from remembered guidance.
2. Update the real classroom review card to append that handoff to the preview launch href.
3. Parse the handoff in `ClassroomV2Preview` and show an entry banner that explicitly acknowledges start / continue / re-check mode.
4. Initialize the preview runtime from the handoff so the next unconfirmed beat is already visible on entry for partial passes.
5. Keep review checklist behavior intact and update planning/state docs.
6. Validate with `npm run build`.

## Validation
- With no remembered memory, launch behaves like a start flow and preview says it is starting fresh.
- With partial memory, the launcher carries a handoff and preview opens with the next unconfirmed beat already visible plus resume framing.
- With complete memory, preview opens framed as a re-check / confidence pass.
- Existing return path still drops back to the legacy classroom route.
- Review checklist still updates exactly as before.
- `npm run build` passes.
