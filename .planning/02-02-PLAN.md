# 02-02 Plan — Layered board primitives and deterministic teacher renderer

Owner: Maximus
Executor: Rex
Status: Completed
Created: 2026-04-04

## Goal
Land the first reviewable V2 layered board slice behind the canonical whiteboard boundary chosen in 02-01, with explicit teacher/student separation and a deterministic teacher renderer driven by tutor actions.

## Scope
1. Add thin board primitives for the teacher layer that can render from `TutorAction[]`.
2. Support the core 02-02 action set reviewably: `board.clearTeacherLayer`, `board.writeProblem`, `board.writeText`, `board.highlight`, `board.arrow`, `board.underline`, and `board.stepBox`.
3. Add an explicit layered board component with:
   - programmatic teacher layer
   - isolated student drawing layer
   - student clear affecting only student ink
4. Dock the new layered renderer into the existing canonical whiteboard boundary without forcing a legacy runtime cutover.
5. Expose the slice inside the existing V2 preview seam so it is easy to review.
6. Verify with `npm run build`.

## Out of Scope
- submit/check evidence packaging and snapshot transport (02-03)
- moving live `VoiceSession` onto the layered renderer by default
- voice/orchestrator runtime integration work from Phase 3
- re-opening the 02-01 whiteboard architecture decision

## Acceptance Checks
- Reviewers can see a teacher layer rendered from tutor actions inside the V2 preview seam.
- Teacher and student layers are explicit in code and visually separate at runtime.
- Student clear does not destroy teacher content.
- Legacy classroom behavior remains intact by default.
- `npm run build` passes.
