# 03-03-PLAN — Normalize student voice evidence into tutor-review inputs

## Goal
Make spoken student answers produce structured, reviewable V2 evidence under `src/lib/classroom-v2/`, analogous to the existing submit/check board packaging, without cutting over the default classroom runtime.

## Scope
- Add a narrow V2 voice-evidence normalization seam under `src/lib/classroom-v2/`
- Package student spoken-answer transcripts into tutor-review-ready evidence with retained turn context
- Keep the shape compatible with existing `StudentEvidence` / tutor-review flow
- Update the V2 preview so reviewers can submit a spoken-answer transcript and inspect the normalized artifact
- Update planning/state docs and run `npm run build`

## Explicitly Out of Scope
- No full browser realtime/runtime cutover
- No legacy `VoiceSession` refactor
- No parent dashboard work
- No whiteboard redesign
- No unrelated cleanup

## Intended Code Shape
1. Extend V2 student voice evidence types to carry normalized transcript data plus review context.
2. Add V2 review helpers that:
   - normalize raw student transcript text,
   - package voice submissions with lesson/turn context,
   - optionally merge voice + board evidence into one tutor-review payload.
3. Update the V2 preview to exercise the seam without changing default classroom behavior.
4. Preserve existing board submission packaging and keep all work under the V2 seam.

## Success Criteria
- Spoken student answers can be represented as structured V2 review evidence
- Voice evidence includes normalized transcript data plus tutor-review context
- Voice + board can coexist in a combined review payload when both are present
- Legacy classroom behavior remains unchanged by default
- `npm run build` passes
