# 05-04-PLAN — Board-surface readability polish in the isolated V2 preview seam

## Goal
Make the isolated V2 preview’s board read more like the center of a real lesson by sharpening board title/subtitle hierarchy, making teacher-vs-student ownership more obvious, and adding a subtle active-surface focus treatment during the main lesson beats.

## Scope
- Stay narrowly focused on the isolated preview seam and canonical layered whiteboard presentation layer.
- Improve board-surface readability without changing runtime/backend/orchestrator contracts.
- Add clearer board ownership copy and visual framing for teacher-owned vs student-owned beats.
- Add a lightweight active-board focus treatment so the board feels like the lesson anchor during rendering, speaking, answering, and review.
- Preserve all existing debug / inspection surfaces.

## Planned implementation
1. Update `src/features/classroom-v2/ClassroomV2Preview.tsx` to derive a board-surface presentation snapshot with:
   - stronger board headline / subheadline copy
   - teacher-vs-student ownership labels
   - active-zone emphasis copy tied to the current classroom state
2. Pass that narrow presentation data into the canonical layered board seam instead of only a generic title/subtitle.
3. Update `src/components/classroom-whiteboard/CanonicalWhiteboard.tsx` and `src/components/classroom-whiteboard/LayeredBoard.tsx` so the reviewable V2 board surface can show:
   - a stronger title/subtitle hierarchy
   - a teacher/student ownership chip
   - subtle active-board framing and focus copy
   - a lightweight teacher-vs-student surface split treatment that stays browser-local
4. Keep the changes additive and review-only. Do not change legacy `VoiceSession` defaults, persistence, DB schema, parent dashboard behavior, or backend/runtime contracts.
5. Write `.planning/05-04-RESULT.md`, update `.planning/STATE.md`, and run `npm run build`.

## Out of scope
- Legacy classroom cutover
- Persistence / DB / backend contract changes
- Parent dashboard changes
- Broad animation or motion systems
- Whiteboard behavior changes beyond readability / presentation polish
