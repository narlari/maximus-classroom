# 02-01 Plan — Freeze a single whiteboard architecture

Owner: Maximus
Executor: Rex
Status: Completed
Created: 2026-04-04

## Goal
Choose the single whiteboard path V2 should carry forward, make that choice explicit, and remove the repo-level ambiguity created by the current dual-path setup.

## Chosen Architecture
Carry forward the active canvas whiteboard path that currently powers `VoiceSession`, and treat it as the canonical pre-02-02 surface.

Why this path:
- it is the live classroom path already in use
- it is the narrowest reversible seam that preserves current behavior
- it creates a stable docking point for 02-02 layered teacher/student primitives without deepening transcript-driven inference work

## Scope
1. Add an explicit canonical whiteboard boundary around the active canvas path.
2. Update live classroom wiring to use that boundary and label emitted whiteboard metadata accordingly.
3. Mark the unused Excalidraw/transcript-inference path as deprecated so it no longer reads like an equal V2 candidate.
4. Reflect the decision in planning/state docs.
5. Verify with `npm run build`.

## Out of Scope
- 02-02 layered board primitives and renderer
- replacing the active canvas implementation with a new layered board
- transcript-driven logic expansion
- teacher/student layered rendering behavior changes

## Acceptance Checks
- Another engineer can inspect the repo and identify one canonical whiteboard architecture for V2.
- Legacy classroom behavior remains the default working path.
- The old transcript-inference whiteboard path is visibly non-canonical.
- `npm run build` passes.
