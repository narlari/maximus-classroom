# 05-02-PLAN — Teacher write/reveal pacing polish in the V2 preview seam

## Goal
Make the isolated V2 preview feel more like a live teacher turn by adding lightweight, browser-local pacing/reveal cues across tutor rendering, tutor speaking, and tutor reviewing, while keeping the work review-only and isolated from the legacy runtime.

## Scope
- Stay primarily inside `src/features/classroom-v2/ClassroomV2Preview.tsx`.
- Add a reviewable notion of teacher-turn progress/pacing that stages the current turn instead of only showing static state labels.
- Improve readability of lesson flow without removing the technical inspection surfaces already added in 05-01.
- Keep all behavior local/demo-level with no backend, schema, persistence, dashboard, or orchestrator contract changes.

## Planned implementation
1. Add a local preview pacing model that derives a staged teacher-turn progress signal from:
   - current classroom state
   - active tutor turn/actions
   - elapsed time since the current preview phase began
2. Surface that pacing model in the child-facing preview with:
   - a simple lesson-flow progress rail
   - a teacher-turn phase label / percentage / “what is revealing now” cue
   - staged copy that differentiates tutor rendering, tutor speaking, and tutor reviewing more clearly
3. Add a reviewable “teacher reveal” panel near the board that explains which part of the turn is currently being revealed and what comes next.
4. Keep all existing debug/inspection panels intact, but enrich the turn-plan review so reviewers can see lightweight per-action reveal status during teacher-owned states.
5. Update `.planning/05-02-RESULT.md` and refresh `.planning/STATE.md` after implementation.
6. Run `npm run build` and only finish if it passes.

## Out of scope
- Legacy `VoiceSession` default changes
- DB/schema/persistence changes
- Parent dashboard changes
- Broad orchestrator API redesign
- Full animation/handwriting systems or runtime cutover
